# Trader Performance vs Market Sentiment

Analysis of how Bitcoin Fear/Greed sentiment relates to trader behavior and
performance on Hyperliquid, prepared for the Primetrade.ai Data Science Intern
assignment.

## Contents

```
.
├── analysis.ipynb          # Main notebook (Parts A, B, C + bonus)
├── analysis.py             # Same analysis as a plain script (jupytext percent format)
├── WRITEUP.md               # Methodology, insights, strategy recommendations (Parts B & C)
├── data/
│   ├── historical_trader_data.csv
│   └── fear_greed_index.csv
├── charts/                  # PNG charts generated by the notebook/script
└── output_tables/           # CSV summary tables generated by the notebook/script
```

## How to run

```bash
pip install pandas numpy matplotlib scikit-learn jupyter
jupyter notebook analysis.ipynb        # run all cells
# — or, equivalently —
python analysis.py                     # regenerates charts/ and output_tables/
```

Both entry points read the same two files under `data/` and produce the same
charts and tables — pick whichever format you prefer to review.

## What's inside

- **Part A** — load + validate both datasets (shape, missing values, duplicates),
  parse timestamps, align by calendar day.
- **Part B** — sentiment-vs-performance comparison, behavior-shift analysis
  (frequency, size, long/short bias), and three reproducible trader segments
  (position size, activity, consistency) plus a KMeans cross-check.
- **Part C** — two evidence-based rule-of-thumb strategies (in `WRITEUP.md`).
- **Bonus** — a simple next-day-profitability classifier (reported honestly,
  including where it *doesn't* beat chance) and a 4-cluster trader archetype
  breakdown.

## Key caveat

The trader data does not include an explicit `leverage` column. Wherever the
original brief references leverage, this analysis substitutes trade size (USD)
and position-size behavior as the closest available proxy — flagged explicitly
in `WRITEUP.md` rather than silently invented.
